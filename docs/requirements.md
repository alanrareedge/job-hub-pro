# Job Hub Pro Requirements

## MVP Goal

Build a simple, mobile-first job operating system for UK trades businesses.

The first version must allow a business to take one job from enquiry through to completion, sign-off and payment tracking.

## Core Users

1. Business Owner
2. Office Administrator
3. Operative
4. Customer

## MVP Workflow

1. Create customer
2. Add property
3. Create job
4. Schedule job
5. Assign operative
6. Operative views job on phone
7. Operative adds notes
8. Operative uploads photos
9. Operative records materials used
10. Customer signs off job
11. Job marked complete
12. Payment requested or marked as paid
13. Full job history retained

## Core Screens

### Dashboard
Shows:
- Today's jobs
- Jobs needing attention
- Outstanding payments
- Recent activity

### Customers
Allows:
- Add customer
- View customer
- Edit customer
- See all properties
- See all jobs

### Properties
Allows:
- Add property
- Link property to customer
- View job history for property

### Jobs
Allows:
- Create job
- Assign customer
- Assign property
- Set status
- Set date
- Assign operative
- Add notes
- Upload photos
- Record materials
- Capture sign-off
- Mark complete
- Track payment

### Operative Mobile View
Shows:
- Today's assigned jobs
- Customer name
- Property address
- Job notes
- Access notes
- Upload photo button
- Add note button
- Materials used
- Complete job button
- Customer sign-off button

### Payments
Allows:
- Set job value
- Record deposit
- Record balance due
- Mark paid
- View unpaid jobs

### Activity Log
Records:
- Customer created
- Job created
- Status changed
- Photo uploaded
- Note added
- Sign-off captured
- Payment updated

## Out of Scope for MVP

- Deep Go High Level integration

- Supplier portal
- Advanced stock control
- Route planning
- Complex reporting
- AI features
- Native mobile app
